# 指定医レポート作成アプリ — 操作マニュアル

## 1. 起動

```bash
cd shitei-report-app
npm run tauri dev
```

またはビルド済みアプリを起動:

- macOS: `指定医レポート作成.app`
- Windows: 配布 ZIP 内の `指定医レポート作成_インストーラー.exe` でインストール後、デスクトップのアイコンから起動

### Windows へのインストール（初めて使う方）

配布された `指定医レポート作成_Windows版.zip` をお使いください。詳しい手順は [windows-install-guide.txt](windows-install-guide.txt) を参照。

1. ZIP を右クリック →「すべて展開」
2. `指定医レポート作成_インストーラー.exe` をダブルクリック
3. 画面の指示に従い「次へ」→「インストール」→「完了」
4. デスクトップの「指定医レポート作成」から起動

## 2. 新規作成

1. 「新規」または Cmd/Ctrl+N
2. 第1〜5症例タブで症例を切り替え
3. 7ステップで入力

## 3. 保存

| 操作 | 説明 |
|------|------|
| 保存 | 初回は名前を付けて保存、2回目以降は上書き |
| 名前を付けて保存 | 常に別名保存 |
| Cmd/Ctrl+S | 保存ショートカット |

ファイル形式: `.shitei`（JSON）

## 4. バージョン履歴

1. ヘッダーの「履歴」をクリック
2. メッセージを入力して「スナップショット保存」
3. 任意の時点で「復元」（復元前に自動バックアップ）
4. JSON書出/読込でUSB移行可能

## 5. チェックリスト

右パネルに55項目を常時表示。ウィザード入力から自動判定（緑/黄/赤）。
手動確認項目はチェックボックスで上書き可能。

### チェック項目から入力欄へジャンプ（v1.3）

- 項目名をクリック（チェックボックス以外）で該当 Step・フィールドへ移動
- 手動確認項目はクリックで案内トーストが表示されます
- 5症例共通の未達項目は、未入力の症例のうち早い番号へ自動切替

## 6. 出力

Step 7 で:
- **テキスト草案** — 入力データ確認用
- **この症例を docx 出力** — Word 下書き（v1.2）
- **様式3-1プレビュー** — 印刷/PDF用

ヘッダー:
- **5症例 docx 出力** — 全症例を一括で Word ファイル生成（v1.2）

### docx 出力の注意

- チェック未達があっても警告後に出力できます
- 空症例は一括出力時にスキップするか選べます
- 出力後、エクスプローラー（Windows）/ Finder（macOS）で保存先フォルダが開きます
- 配布用ビルド前に sidecar ビルドが必要です
  - macOS: `npm run build:sidecar`
  - Windows: `npm run build:sidecar:win`

### 入院形態・症例別ヒント（v1.3）

- Step 1/3/5/6 は入院形態（措置/医療保護）に応じて記載要件が表示されます
- Step 4/5 は症例カテゴリ（F0〜F9）に応じたヒントが表示されます
- Step 6 には法的手続確認リスト（様式3-1 への✓記入参照）が表示されます

## 7. 旧データの移行

1. 旧 `shitei-report-wizard.html` で JSON 保存
2. 「旧JSON取込」
3. `.shitei` として保存

## 8. 最近使ったファイル

「最近のファイル」から直近10件を番号選択で開けます。

## 9. 自動リカバリ

編集中は500msごとに自動保存。異常終了後、次回起動時に復元ダイアログが表示されます。

## 10. 共通項目の一括反映（v1.1）

1. 任意の症例の Step 1 で申請日・医療機関・指導医を入力
2. 「空欄へ一括反映」ボタンをクリック
3. 他4症例の空欄のみ自動入力（既存の入力は保護される）

## 11. 入力の快適性（v1.1）

- 長文入力中もカーソル位置が維持されます
- 右側チェックリストは入力停止から約0.5秒後に更新されます

## 12. 元に戻す・やり直し（v1.4）

- ヘッダーの「元に戻す」「やり直し」、または Cmd/Ctrl+Z / Cmd/Ctrl+Shift+Z
- 入力停止から約0.8秒後に1 undo 単位として記録されます
- 共通項目一括反映・Step コピーも undo 対象です

## 13. 他症例へ Step コピー（v1.4）

1. Step 2〜7 の「他症例へコピー」をクリック
2. コピー先症例をチェック（現在の症例以外）
3. 「反映」— 選択症例の**空欄のみ**にコピー（既存入力は保護）
4. Step 1（表紙・ICD・入院形態）は症例固有のためコピー対象外
